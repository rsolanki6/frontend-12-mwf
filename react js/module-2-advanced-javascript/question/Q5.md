# What is undefined?

`undefined` is a **primitive data type** in JavaScript that indicates a variable has been declared but **has not been assigned a value**.

## When Does `undefined` Occur?

### 1. Declared but not Assigned
```javascript
let x;
console.log(x); //undefined
```

### 2. Missing Function Return Value
If a function does not return anything, it returns `undefined`.
```javascript
function test() {}
console.log(test()); // undefined
```

### 3. Non-existent Object Properties
Accessing a property that does not exist return `undefined`.
```javascript
let obj = {name: "JS"};
console.log(obj.age); // undefined
```

### 4. Function Parameters Not Passed
```javascript
function greet(name){
     console.log(name);
}
greet(); //undefined
```

`undefined` VS `null`
| undefined | null |
|-----------|------|
| Automatic value | Assigned intentionally |
| Means "not defined yet" | Means "empty value" |
| Default state | Explicitly set |

### Important Notes
- `undefined` is a global property
- You should not manually assign `undefined`
- Use `null` when you want to intentionally clear a value
